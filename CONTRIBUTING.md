# Contribution Guidelines

Thank you for your interest in contributing to **Unilang**! We welcome contributions from everyone.

## How to Contribute

1. **Fork the Repository**: Use the fork button to create your own copy of the repository.
2. **Create a Branch**: Create a new branch with a descriptive name relevant to your work.  
   ```bash
   git checkout -b your-feature-branch
   ```
3. **Make Your Changes**: Implement your changes and commit them with clear, concise commit messages.
   ```bash
   git commit -m "Your message here"
   ```
4. **Push Your Changes**: Push your changes back to your forked repository.
   ```bash
   git push origin your-feature-branch
   ```
5. **Open a Pull Request**: Navigate to the original repository and create a pull request explaining your changes.

## Guidelines

- Be respectful and considerate in discussions.
- Keep the code style consistent with the rest of the project.
- Write clear, concise commit messages.
- Test your changes thoroughly before submitting.

## Reporting Issues

If you find a bug or have a feature request, please open an issue in the repository with as much detail as possible.
